# 質問セットの品質チェック

生成された $Q$ に対し 3 指標を計算し、質問設計が良いかを確認する。`quality-check` はケース集合に対する各質問の $f_E$ 行列（0/1）を受け取り算出する。

## 1. Yes-rate spread（はい率の分散）

各質問 $q_i$ の「はい」率 $p_i = \frac{1}{|\text{cases}|}\sum_j f_E(x_j,y_j,q_i)$ を出し、次元内の **spread = max$_i p_i$ − min$_i p_i$** を見る。

- 良い: 次元内で $p_i$ が広く分散（spread が大きい）。異なる難易度を捕捉している。
- 悪い: 全質問がほぼ同じ率（spread≈0）→ 質問が冗長で情報を足していない。
- 出力: 各 $p_i$、min/max、spread。

## 2. Inter-question correlation（質問間相関 φ 係数）

同一次元内の質問ペアの **phi 係数** を算出。2×2 分割表（両質問の 0/1）から:

$$\phi = \frac{n_{11}n_{00} - n_{10}n_{01}}{\sqrt{(n_{11}+n_{10})(n_{01}+n_{00})(n_{11}+n_{01})(n_{10}+n_{00})}}$$

- 良い: **低相関**（$|\phi|<0.3$ のペアが多い）。各質問が異なる側面を捉え、集約による分散削減が効く。
- 悪い: 高相関。冗長で分散削減が効きにくい。
- 出力: 次元ごとの平均 φ、$|\phi|<0.3$ の弱相関ペア割合、全体平均 φ。

## 3. Pairwise coverage（失敗モードのカバレッジ）

質問ペアが **異なる失敗モード** をカバーしているか。ほぼ無相関(φ≈0)かつ **異なる yes 率** を持つペアは、別々の失敗を捉えている。

- 判定: 各ペアで「φ が低い AND yes 率差が一定以上」を満たす割合を「カバレッジ良好ペア率」として出す。
- 高いほど、全体判断が見落とす失敗パターンを分解が拾えている。

## 判定と対処

| 兆候 | 意味 | 対処 |
|------|------|------|
| spread がほぼ 0 | 質問が冗長・情報を足していない | メタプロンプトで質問を絞る / 違反例を鋭くする |
| 平均 φ が高い(>0.5) | 分散削減が効きにくい | 質問を異なる失敗モードに分割し直す |
| coverage 良好ペア率が低い | 失敗モードを広く拾えていない | 次元を増やす / 質問を多様化 |

> 人間ラベルがある場合は、次元スコア $S_d$ と人間スコアの相関（Spearman ρ / Kendall τ）も併せて見ると最良。無ければ上記 3 指標で代替する。
