# 表現力スキル

「読み手にハッとさせて、『そう来たか』『おもしろい』『気になる』という刺激を与える情報設計・空間設計・体験設計」を導くための思考スキル。

デザイン・UX・文言・キャッチコピー・体験設計などを考える際に、安易な"普通の発想"に流されず、知覚 → 組み替え → 表現 のプロセスを反復して**複数案を提案する**ことを目的とする。

---

## 基礎: 良い表現の前提

良い表現を考える前に、以下を必ず意識する。

- **【最優先】対象（ペルソナ・文脈）に沿った言葉遣いをする。** これが他のどの技法よりも重要。そのプロジェクトのターゲットが個人（B2C / 個人開発のエンドユーザー）なのか B2B SaaS（導入担当者・決裁者・現場ユーザー）なのかをまず見極める。ペルソナの立場・温度感・語彙・期待する信頼感、そしてプロジェクトの文脈（ブランド・利用シーン・読まれる場所）に合致した言葉遣いを選ぶ。同じ「ハッとさせる表現」でも、刺さる相手・適切な語り口・許される崩し方は対象によって変わる。**以下の『ハッとさせる』技法（倒置・真逆・崩しなど）はすべてこの文脈適合の上に乗るもの**であり、文脈に合わない奇抜さは即座に捨てる。前提を取り違えると、表現の巧拙以前に的を外す
- **足さない。むしろ減らす。** やらないことを決めて、本当に必要なことにフォーカスして伝える
- **余白を持たせる。** 読み手に考えさせる隙間を残す
- **普通の発想は人の心を動かさない。** よくある構成・テンプレ的な訴求は刺激にならない
- **少し引っかかる。** 予想外の展開 / 僅かな余白が「気になる」を生み出す
- **すべての情報を文字にすればよいわけではない。** 説明しすぎない勇気を持つ

> ❌ やりがちな失敗: 情報を詰め込む / 説明的にする / 普通の言い回しで安心する
> ✅ 目指す状態: 心地よいズレ / 想像の余白 / 一見ハッとする違和感

---

## 思考プロセス: 3 つのフェーズ

**重要**: 以下は直線的な手順ではない。**反復的に複数の思考を行き来し**、具体と抽象、疑問と答えを往復することで導き出す。
**必ず複数の思考ルートを走らせ、複数案を提案する。**

### Phase 1: 知覚する (Perceive)

対象を多角的に観察し、違和感や引っ掛かりの種を見つける。

| アプローチ | 内容 |
|---|---|
| **ユーザーへのヒアリング（実現したいこと / 世界観）** | ユーザーにヒアリングをして、実現したいこと、作りたい世界観を正しく理解する。前提を取り違えると表現の方向ごとズレるため、知覚の起点として最優先で行う |
| **「なぜ作ろうと思ったか」のしつこい深掘り** | なぜこのサービスを作ろうと思ったのかをしつこく質問して把握する。表層の理由の裏にある原体験・違和感・コンプレックスなど、思わぬバックグラウンドにこそ表現のヒントがある。1 回の回答で止めず「それはなぜ？」を重ねる |
| **相手の心の位置の想像** | ユーザーが今どんな気分・文脈にいるかを想像し、違和感を意図的に設計する |
| **視点の逆転** | 当たり前と言われているものを「もしも逆だったら」という発想で見つめてみる |
| **異分野からの転用** | 取り扱っている分野が、他の全く異なる分野と共通項がないかを見出し、そこからインスピレーションを膨らませる |
| **角度を変えて見る** | 同じ対象を別の側面・別の時間軸・別のスケールで見直す |
| **なりきる** | ユーザー、もしくはユーザーのリアルな 1 つの生活シーンを深く想像する |
| **常識を疑う** | 「これって本当に必要？」「なぜこれが当たり前？」を何度も問い直す |

### Phase 2: 組み替える (Recombine)

知覚した要素を組み合わせ・組み替えて、意外性を作る。

| アプローチ | 内容 |
|---|---|
| **既存概念の組み合わせ** | 既存の概念を組み合わせ／組み替えて意外性を生む（例: どこか古めかしいけど新しい / 速くあるべきなのに遅い） |
| **量と多角ブレスト → 接続 → 世界観** | 圧倒的な量と多角度のブレスト → 気になる要素の繋ぎ合わせ → 新しい世界観を作る |
| **意外なもの倒置法** | 「&lt;X&gt; を &lt;Y&gt; に」というコンセプト表現。心地よいズレを端的に表すキャッチコピーとして機能する。**ただし多用厳禁** — 倒置・体言止めは「それっぽさ」が出るぶん思考の逃げ道になりやすく、多用すると陳腐化する。使うなら複数案中の 1 案までに留め、デフォルトの語順・文体にしない |
| **マイナス → プラスへの転換** | マイナス要素をプラス思考で捉え直し、新たな意味を見出す（例: 遅い → 待つ時間がある贅沢） |
| **真逆思考** | 真逆を思考してみて組み合わせる（甘いの反対は？ 優しいの反対は？ 便利の反対は？） |
| **ダジャレ的思考** | 音や言葉の重ね合わせを利用する（例: ココカラファイン / ラクスル） |

### Phase 3: 表現する (Express)

知覚して組み替えたものを、より多くの人に伝わるように**翻訳**する。

| アプローチ | 内容 |
|---|---|
| **文脈の置き換え** | 近くの組み合わせを違う文脈に置き換えて新しい意味を見出す |
| **自分の感じたこと → 相手が感じることへ翻訳** | 表現とは、自分の感覚を相手の感覚に翻訳できるかが問われる |
| **"普通"より少しズラす** | 普通から少しだけズラした伝え方をする（外れすぎは違和感が多すぎる） |
| **順番と感情設計** | 伝える順番を工夫し、完璧ではないアウトプットを見せることで心地よいズレを生む。逆転すればよいわけではなく、どういう感情設計をするのかを問う |
| **語らない勇気 / 余白** | 説明しすぎない。想像できる余白を持つからこそ、興味を持ち理解が深まる |

## 提案の出力フォーマット

表現を提案するときは、必ず以下の構造で複数案を出す。

```
## 表現案

### 案 A — [一言で表すコンセプト]
- 知覚: [何に気付いたか / どの視点を取ったか]
- 組み替え: [どう組み合わせ・倒置・反転したか]
- 表現: [具体的なコピー / UI / 体験]
- 狙うズレ: [読み手にどんな『おっ』を生むか]

### 案 B — [一言で表すコンセプト]
（同上）

### 案 C — [一言で表すコンセプト]
（同上）

---

### 思考ログ（任意）
- どの「常識」を疑ったか
- どの「真逆」を試したか
- 不採用にした方向性とその理由
```

最低でも **3 案**を出し、それぞれが**異なる思考ルート**から生まれていること。似た案を並べるだけでは意味がない。

## ガードレール: やってはいけないこと

| ❌ 避ける | ✅ 代わりにやる |
|---|---|
| ターゲットを問わず同じ語り口で表現する | ペルソナ・文脈（個人 / B2B SaaS）を見極め、語り口・温度感・語彙を合わせる（最優先） |
| 倒置法・体言止めに安易に流れる | まず通常の語順で刺さる表現を探す。倒置はあくまで一つの選択肢で、デフォルトにしない |
| 情報を全部盛り込む | 削る・余白を残す |
| 「普通」「無難」「定番」に逃げる | 少しだけズラす（外しすぎない） |
| 説明的・解説的なコピー | 想像の余白を残す |
| 1 案だけ出す | 複数の思考ルートから複数案を出す |
| 逆張りすればいいと思い込む | 感情設計として何が心地よいズレかを問う |
| 思いつきだけで提案 | 知覚 → 組み替え → 表現 のプロセスを踏む |
