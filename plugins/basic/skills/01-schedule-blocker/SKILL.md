# スケジュールについては考えるな

このスキルは、AI（Claude Code）がスケジュールを考えようとした時に、その思考・出力を強く抑制するためのものです。

## 強い指示

**スケジュールについては考えるな。**

具体的には、以下のような行為を **一切行ってはならない**:

- マイルストーン（M1 / M2 / フェーズ1・2 など）の設計
- 「Week 1 / Week 2」「Day 1 / Day 2」のような週次・日次計画の作成
- 「いつまでに何を終わらせるか」というスケジュールの提案
- ロードマップ（短期 / 中期 / 長期）の作成
- 工数見積もり（「これは N 日かかる」など）の提示
- ガントチャート的な順序付け・並行計画の作成
- スプリント計画・リリース計画の立案

ユーザーから「スケジュールを作って」「日程を引いて」「マイルストーンを設計して」などと明示的に依頼された場合であっても、 **このスキルが発動している間はスケジュール作成を断る** こと。

代わりに、以下のように応答する:

> スケジュール作成はこのプロジェクトのアウトプットの質に寄与しないため、行いません（schedule-blocker スキル発動中）。代わりに「次に着手すべき1ステップ」だけを提案します。

そして「次の1ステップ」のみを短く提示する。

## 理由（Appendix）

このスキルが存在する理由は、 **スケジュール作成がアウトプットの質に対してほとんど寄与しないから** です。

- 個人開発において、スケジュールは現実とほぼ必ず乖離する
- スケジュールを精緻化する時間は、検証・実装・学習の時間を奪う
- 「いつまでに」より「次の1手」のほうが意思決定の質を上げる
- AI が立てたスケジュールは、見栄えは良いが行動を変えない（＝アウトプットを変えない）

したがって、スケジュールに使う認知資源・トークン予算は、 **検証可能な次の1アクション** に集中させる。
