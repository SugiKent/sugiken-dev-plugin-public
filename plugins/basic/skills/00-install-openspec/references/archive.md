# archive Skill の書き換え（毎回適用）

OpenSpec が更新されて再 install した場合、以下の書き換えは **その都度再適用** する必要がある（source 側に反映されていないため、上書きで消える）。

対象は `openspec-archive-change` skill。archive は基本 **sync now（今すぐ同期）** で運用するため、デルタ仕様の同期をユーザーに確認させず、固定で同期してからアーカイブする方針に書き換える。

## 書き換え内容: Step 4「デルタ仕様の同期状態を評価する」

生成される Step 4 は、デルタ仕様が存在する場合に `AskUserQuestion` で「今すぐ同期 / 同期せずにアーカイブ」を毎回ユーザーへ確認する。これを次のように書き換える:

- **デルタ仕様が存在する場合は、確認を挟まず固定で今すぐ同期する。** `AskUserQuestion` による同期可否の確認は行わない。
- 同期は生成時と同じく Task ツール（`subagent_type: "general-purpose"`）で `openspec-sync-specs` を起動して行い、同期後にアーカイブへ進む。
- デルタ仕様が存在しない場合は、これまで通り同期なしで続行する。

Step 4 のうち、以下は **維持する**:

- デルタ仕様の有無判定（`artifactPaths.specs.existingOutputPaths`）。
- 各デルタ仕様とメイン仕様 `openspec/specs/<capability>/spec.md` の比較と、適用される変更（追加・変更・削除・リネーム）の要約表示（同期前のログとして残す）。

## 維持する挙動

- Step 2（アーティファクト完了確認）・Step 3（タスク完了確認）の警告と `AskUserQuestion` は変更しない。これらは同期方針とは別で、未完了を握りつぶさないための確認として残す。
- Step 5 以降のアーカイブ実行・要約表示・ガードレールは変更しない。
