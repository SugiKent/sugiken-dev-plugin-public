# MVP.md の構築方法

- はじめから完全なドキュメントにする必要はない
- まずはブレスト的に複数の選択肢を MVP.md に書くことも重要である。そこから人間が判断し選択する。
- 合意できた内容については「確定」と分かりやすくマークしておく。
- 履歴をドキュメント下部に書く（内容と理由をセットで書き溜める）

# MVP.md の構造

docs/MVP.md の最終的なあるべき姿は以下のフォーマットについて人間と合意された結果がまとまること

```
# コアな価値

# 解決する顧客のジョブ

# ビジネスモデル

# MVP で検証すること

# マーケティング戦略

# PLG戦略

# 履歴

```

逆にスケジュールやコスト見積もりなどは不要
